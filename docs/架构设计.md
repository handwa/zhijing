# 知境 - 架构设计文档

## 项目概述

知境是一款基于端侧AI的隐私保护型情境感知知识助理，采用"认知型"而非"执行型"路线。

## 整体架构

```
感知层（多源采集）→ 理解层（端侧AI）→ 关联层（知识图谱）→ 推送层（智能推荐）
```

## 技术栈

| 模块 | 技术 |
|------|------|
| Android端 | Java + Room + TFLite |
| 后台服务 | Spring Boot 3 + MySQL 8 + MyBatis-Plus |
| 端侧AI | TensorFlow Lite 2.x |
| 网络通信 | Retrofit2 + OkHttp |
| 依赖注入 | Hilt |
| 异步处理 | WorkManager + ExecutorService |

## 数据库设计

### Android端（Room/SQLite）

**knowledge_entity 知识实体表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 主键 |
| name | TEXT | 实体名称 |
| type | TEXT | 类型(人物/作品/事件/概念) |
| weight | REAL | 权重/重要度 |
| created_at | INTEGER | 创建时间戳 |

**knowledge_relation 知识关联表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 主键 |
| from_entity_id | INTEGER | 起始实体 |
| to_entity_id | INTEGER | 目标实体 |
| relation_type | TEXT | 关系类型 |
| strength | REAL | 关联强度 |

**user_content 用户内容表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 主键 |
| content | TEXT | 原始内容 |
| source_app | TEXT | 来源应用 |
| keywords | TEXT | 提取的关键词(JSON) |
| created_at | INTEGER | 时间戳 |

### 后台（MySQL）

**user 用户表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| device_id | VARCHAR | 设备唯一标识 |
| created_at | DATETIME | 注册时间 |

**sync_record 同步记录表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| user_id | BIGINT | 用户ID |
| data_type | VARCHAR | 数据类型 |
| synced_at | DATETIME | 同步时间 |

## 核心流程

### 用户分享内容流程
1. 用户从任意App分享文本 → 触发 ShareReceiverActivity
2. 文本送入 TFLiteEngine 提取实体和关键词
3. 实体写入 Room 本地知识图谱
4. RecommendEngine 在图谱中检索关联内容
5. 构建推送卡片（含可解释推导链）展示给用户

### 推送卡片交互
- 点击"展开" → 生成知识整合摘要
- 点击"为什么" → 展示可视化推导链
- 点击"稍后" → WorkManager延迟推送

## 隐私设计原则

- 所有AI推理在本地完成，不上传原始内容
- 云端仅同步匿名化的实体ID和关联强度
- 用户可随时清空本地知识图谱
